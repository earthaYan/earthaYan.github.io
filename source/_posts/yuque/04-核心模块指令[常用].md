- daemon:on/off
  - 启用或禁用守护进程模式
  - 禁用则该程序就不能在后台启动，当他在shell下运行时，将留在前端运行     
- error_log    /file/path  level
  - 默认值：logs/error.log error
  - 使用环境：main http server location
  - 级别：debug info notice warn error crit（详细程度由高到低）
- log_not_found:on/off
  - 使用环境：main http server location   
  - 开启或禁用记录404错误
- master_process:on/off
  - 默认值on
  - 开启则nginx将开启多个进程：一个master主进程和worker进程
  - 禁用则Nginx则会以独一无二的进程来运行，
  - 仅被用于测试,客户端无法连接到master进程的服务器
-  pid logs/nginx.pid
  - 用于存放Nginx守护进程的pid文件路径
  - 默认值为编译时定义
- ssl_engine
  - 语法：ssl_engine enginename(系统的有效硬件SSL加速器)                      
- thread_stack_size
  - 语法：thread_stack_size 1m;
  - 默认值None
  - 定义线程堆栈的大小
- timer_resolution
  - 语法：timer_resolution 10s;
  - 控制系统调用gettimeofday()时间间隔
- user
  - 定义用户及其相应的个组
  - Nginx的worker进程将使用这些设置来运行
  - user username;
- worker_threads
  - 语法：worker_threads 9；
  - 为每一个worker进程定义一定数量的线程
- worker_cpu_affinity
  - 与worker_process协同工作
- worker_processes
  - 语法：worker_processes 4；
  - 定义worker进程的数量
  - nginx可以将请求的处理分配到多个worker进程
- worker_rlimit_core
  - 定义每个进程的内核文件大小
  - worker_rlimit_core  100m;
- worker_rlimit_nofile
  - 定义一个worker可以同时处理的文件数量
  - worker_rlimit_nofile 10000；
- working_directory /usr/local/nginx
  - 默认值在配置编译时通过prefix开关选项指定
- **Events模块**
  - accept_mutex 
    - on/off,默认on
    - 启用/禁用使用一个接受互斥（现象）锁来打开套接字监听
  - accept_mutex_dalay
    - 500ms
    - 定义一个worker进程在尝试再次获取资源之前应该等待多长时间
    - ·accept_mutex 有效on时才会使用使用这个命令
  - worker_connections
    - 定义一个worker进程能够同时连接的数量
- **Configure模块**
  - include指令
