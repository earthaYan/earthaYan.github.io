### 错误分类：

- 即时运行错误：代码错误
- 资源加载错误

<a name="TOkfS"></a>
### 捕获方式：

- 即时性错误：try-catch
- window.onerror,一定比其他脚本先执行
- 资源加载错误：object.onerror
- performance.getEntries()获取到所有已经加载的资源
- Error事件捕获：

```javascript
window.addEventListener('error',function(e){console.log(e)},true)
//第三个参数必须是true,表示这是捕获阶段
```
### 注意：
1.  把window.onerror这个代码块分离出去，并且比其他脚本先执行（注意这个前提！）

2.  由于网络请求异常事件不会冒泡，需要在捕获阶段进行处理

3.  不能捕获promise的错误信息

4.  跨域资源需要专门处理，需要在script标签加上crossorigin属性，服务器设置Access-Control-Allow-Origin

5.  window.onerror 函数只有在返回 true 的时候，异常才不会向上抛出，否则即使是知道异常的发生控制台还是会显示 Uncaught Error: xxxxx。

<a name="2sRls"></a>

### 跨域的js错误：

1. 可以捕获到跨域的js错误，但是不会有详细的错误信息，只会显示script error
1. 解决方案：
  1. 在script标签增加`crossorigin `属性
  1. 设置js资源响应头`Access-control-allow-origin:*`


<a name="3LFIF"></a>

### 上报错误的基本原理：

1. Ajax通信的方式上报
1. 利用Image对象上报
```javascript
// 参数为错误信息
function report(error) {
  var reportUrl = 'http://xxxx/report';
  new Image().src = reportUrl + 'error=' + error;
}
```
